# File Processor

Приложение ищет входные файлы по заданной маске, обрабатывает их блоками и сохраняет изменённые файлы в выбранную выходную директорию. Обработка выполняется в отдельном потоке, поэтому интерфейс не зависает даже при работе с большими файлами.

## Возможности
1) Выбор директории для поиска входных файлов
2) Выбор директории для сохранения результата
3) Настройка маски выходных файлов
4) Возможность удалять входные файлы после обработки
5) При совпадении имени выходного файла можно или перезаписать файл, или добавить в название файла счётчик
6) Разовый запуск обработки
7) Работу по таймеру с заданной периодичностью
8) Обработка больших файлов блоками без загрузки файла целиком в память
9) Отображение прогресса текущего файла
10) Отображение общего прогресса обработки
11) Корректное завершение работы приложения при активной обработке

## Архитектура

QML -> AppController -> ProcessManager -> FileProcessor

### FileProcess
Отвечает за обработку одного инстанса файла

Открывает входной файл, открывает выходной файл, читает файл блоками по 8 Мб, применяет операцию XOR,
отправляет сигналы прогресса

### ProcessManager
Управляет очередью файлов на обработку

Получает список файлов на обработку, последовательно запускает FileProcessor

### AppController
Связывает QML и backend

### Сборка
Сборка через vcpkg и cmake происходит с помощью следующей команды:
```
cmake -S . -B build -G Ninja ^
 -DCMAKE_BUILD_TYPE=Debug ^
 -DCMAKE_TOOLCHAIN_FILE=Path_To_vcpkg ^ 
 -DVCPKG_TARGET_TRIPLET=x64-mingw-dynamic ^ 
 -DVCPKG_HOST_TRIPLET=x64-mingw-dynamic
```
Где `Path_To_vcpkg` - путь до `vcpkg.cmake`
